01 / 基础概念

Transformer 是什么——架构 vs 模型

Transformer 是一种神经网络架构（结构设计方案），由 Google 在 2017 年论文《Attention Is All You Need》中提出。它本身不存储任何知识，就像一栋建筑的设计图——图纸不是房子，只规定了房子怎么搭。

关键区别

Transformer 是架构（设计图），大模型是用这个架构经过完整训练流程后得到的产物。GPT-4、Claude、DeepSeek 都用 Transformer，但 Transformer ≠ 大模型。

Transformer 的核心机制是 Self-Attention（自注意力）——处理每个词时，能同时"看到"句子里所有其他词，并判断它们之间的相关程度。这是它碾压之前所有架构的根本原因。

输入文本 → Token → 向量
↓

↓ × N 层

↓

↓
输出预测下一个 token 的概率分布
02 / 核心理解

Transformer 学的是规律，不是内容

训练完的 Transformer 权重里不是原始文本，而是从海量语料中蒸馏出的统计规律和语义关联。

模型不会"记住"某篇文章，但会"知道"医学文本里"诊断"后面常跟哪类词；不会存储"巴黎是法国首都"这句话，但权重里编码了这种关联。

类比

不是复印机（复制内容），是压缩了规律的函数（提取模式）。就像一个读过十万本书的人——他不会逐字背诵，但能用自己的语言重新表达那些知识。

03 / 训练流程

大模型的完整训练流程

「大模型」不是只靠喂数据就完成的，实际是三个阶段叠加的结果：

1

预训练 Pre-training

海量语料（数万亿 token），训练目标极简单：给你前 N 个词，预测第 N+1 个词。重复万亿次，模型被迫学会语言规律和世界知识。结果是一个"基座模型"——博学但不听指令。

2

指令微调 SFT

用高质量问答对（几万～几十万条）继续训练，让模型学会"理解指令 → 给出有用回答"。数据量少但质量要求高。结果是模型变成了"助手"。

3

RLHF / DPO 偏好对齐

让模型的回答更符合人类偏好（有用、无害、诚实）。人工标注员对多个回答排序，训练奖励模型，再用强化学习优化主模型。现代常用更简单的 DPO 替代。

阶段数据量目标结果
预训练万亿 token预测下一个词基座模型，懂语言但不听话
SFT 微调万～十万条理解并执行指令能对话的助手
RLHF/DPO偏好排序数据符合人类价值观有用、安全、诚实的模型
04 / 训练数据

训练数据——不是随便抓取就行

预训练数据确实主要来自互联网抓取（Common Crawl 每月爬取整个互联网，数据量 PB 级），但直接使用会出大问题，中间有关键的清洗步骤。

数据清洗（被严重低估的步骤）

原始网页数据中充斥着重复内容、乱码、广告 HTML 标签、低质量灌水文。Llama 3 最终用了 15 万亿 token，但原始抓取量是这个数字的好几倍，大量数据在清洗阶段被丢弃。

光靠互联网数据还不够，各家会专门混入高质量来源，配比是核心竞争力：

数据来源作用影响能力
书籍质量高、长文本、逻辑严密深度推理、写作
GitHub 代码结构化、逻辑强编程 + 逻辑推理（即使不问代码）
学术论文专业知识密度高STEM、科学
Wikipedia事实密度高、多语言知识广度
网页覆盖广、时效性强常识、时事
05 / 核心机制

模型怎么"无师自通"回答问题

训练时从没有人告诉模型「猫爱吃鱼的原因是……」。模型学的是更底层的东西：

给你前面这些词，下一个词最可能是什么？——仅此而已。任务极其简单，但做了几万亿次之后，为了能准确预测下一个词，模型被迫在权重里压缩进去了大量世界知识。

语料里可能有这些片段散落各处：「猫对腥味很敏感……」「猫是肉食动物，需要大量蛋白质……」「野猫会在河边捕鱼……」

没有一句在"回答问题"，但模型学会了这些词在什么语境下共现。当你问「为什么猫爱吃鱼」，模型用积累的规律重新组装出答案——不是查找，是生成。

人类做了什么

没有人告诉模型任何具体知识。人类提供了语料（天然包含人类知识），并设计了"预测下一词"这个训练目标 + 反向传播机制。模型从来不知道"正确答案是什么"，只知道"我预测错了，权重要往这个方向调一点"。重复万亿次，知识自然沉淀。

06 / 模型形态

训练好的模型长什么样

训练好的模型本质上就是一个巨大的数字文件（或几十个分片）：

# 物理文件大小（近似） GPT-3 ~350 GB Llama 3 70B ~140 GB Claude 3.5 （未公开） # 文件内容 0.0023 -1.4521 0.8834 -0.2156 0.7743 0.3312 -0.0087 1.2341 -0.5521 0.1198 ... 几百亿个这样的浮点数，别的什么都没有

里面没有文字，没有规则，没有代码逻辑——纯粹是数字。

最形象的类比

原始语料是一座图书馆，几百 TB。训练完后，所有"知识"被蒸馏压缩进一个 140GB 的数字文件。原来的书全扔掉了，只剩这个文件。但拿着它去问问题，它能重新"生长"出答案——不是查找，是根据数字间的数学关系重新生成。

每个数字单独看毫无意义。知识不在某一个数字里，而是分散在所有数字的组合关系里——就像大脑里没有一个神经元专门存着「猫爱吃鱼」，但几千个神经元的激活模式组合在一起，就能让你想起这件事。

07 / 参数结构

权重矩阵——几百个，不是一个

参数不是一个大矩阵，而是几百到上千个矩阵，每个矩阵负责不同的变换任务。

以 Llama 3 70B 为例（80 层），每一层有：

位置矩阵作用
Self-AttentionQ、K、V、O 四个矩阵计算词与词之间的注意力
FFN2～3 个矩阵提炼和变换特征
Layer Norm少量参数数值归一化（工程辅助）
输入/输出Embedding 矩阵词 ↔ 向量互转

80 层 × 每层约 7～8 个矩阵 = 约 600+ 个矩阵。每个矩阵本身也不小，比如 Q 矩阵可能是 8192 × 8192，约 6700 万个参数，光这一个就这么大。

推理时，输入文本像水流过筛子——依次流过每一个矩阵，每一层做一次矩阵乘法，最后从另一端流出来一个概率分布，概率最高的词就是输出。整个过程没有任何"查找"，纯粹是数学计算。
